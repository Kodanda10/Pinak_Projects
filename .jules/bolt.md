## 2024-05-24 - NumPy Vector Store Optimization
**Learning:** In a pure-NumPy VectorStore like the one in `app/services/vector_store.py`, calculating L2 distances using 2D matrix multiplications (`np.dot(..., query.T).flatten()`) causes significant overhead and intermediate array allocations. A fully vectorized 1D approach using `ravel()`, 1D `np.dot`, and in-place array math drastically reduces latency. Additionally, iteratively extending large Numpy arrays with `np.vstack()` during individual `add_vectors` calls is O(N^2) and becomes a severe bottleneck. Using a Python list to cache pending vectors inside the `batch_add()` context manager and then concatenating them all at once is orders of magnitude faster.
**Action:** When working with O(N) Numpy similarity search or batch array creation, always use 1D operations and in-place math for queries, and batch array concatenations instead of O(N^2) `np.vstack` loops.
