## 2024-05-18 - Thread-safe Batching Context Manager using threading.local()
**Learning:** Using `yield` within a context manager (`batch_add`) to aggregate data (like vector additions) runs the risk of slow O(N^2) continuous reallocation if we continually append to NumPy arrays. It also requires careful thread isolation, as a shared object instance lock does not protect batched states correctly if inner methods need to run concurrently or re-enter.
**Action:** Implemented `threading.local()` to isolate the state of active batches. This allowed the context manager `batch_add` to cache the vectors, ids, and norms within thread-local Python lists during execution. In the `finally` block, a single `np.concatenate` and `np.vstack` combination is performed, successfully reducing O(N^2) repetitive allocations to O(N) operations, while being 100% thread-safe.
