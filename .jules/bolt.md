## 2025-02-17 - [Add index on embedding_id for Memory Service SQLite]
**Learning:** Found an unindexed linear scan lookup via `embedding_id IN (...)` resulting in ~18ms latency for 100k rows. `DatabaseManager.get_memories_by_embedding_ids()` searches for specific matches across three tables (`memories_semantic`, `memories_episodic`, `memories_procedural`). Adding an index on `embedding_id` to these three tables in `_init_db` reduced the search latency to 0.08ms (a 200x speedup). This architecture lacks indexes on frequently filtered fields and doesn't explicitly add index generation for custom relationships.
**Action:** When working on DB abstractions (`app/core/database.py`), add indexes explicitly during table creation if the table interacts with hybrid search via the `embedding_id`. Also, always update test cases like `test_doctor_backfill.py` mock `_init_db` definitions to align with real schema changes.
