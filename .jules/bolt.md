## 2024-05-18 - [Optimize NumPy VectorStore Distance Calculation]
**Learning:** In the fallback NumPy `VectorStore` (used to avoid FAISS segfaults), calculating L2 distances sequentially (`.flatten()` on matrix-vector products) and allocating new arrays for intermediate steps (`sq_dists = norms + q - 2*dot`) adds significant overhead during high-frequency searches. Flattening a 2D query to 1D via `.ravel()` allows faster 1D NumPy dot products, and applying operations in-place (`+=`, `-=`, `np.maximum(..., out=...)`) prevents memory allocations.
**Action:** When working with core mathematical bottlenecks in NumPy, heavily scrutinize intermediate array allocations. Use `.ravel()` to ensure 1D shape contexts where applicable, and always pass `out=` parameters or use augmented assignment (`+=`) to reuse existing memory buffers.
