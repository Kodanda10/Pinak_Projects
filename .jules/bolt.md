## 2024-05-24 - [Thread Spam Bottleneck in Debounced Saves]
**Learning:** In the `VectorStore` class, a "debounced" save method was implemented by creating and cancelling `threading.Timer` objects for *every single vector addition*. While mathematically a debounce, in practice, creating and destroying a python thread 10,000 times during a bulk load loop adds massive overhead (~3.0s vs 0.008s for a simple throttler) and lock contention, neutralizing any I/O savings.
**Action:** When implementing debouncing in Python where the triggering event can happen extremely rapidly in a tight loop (like vector inserts), use a throttler that checks `.is_alive()` on the existing timer thread, or use a single dedicated background thread checking a dirty flag/timestamp, rather than creating a new thread per event.
