## 2024-05-15 - [Database Connection Pooling]
**Learning:** Re-creating a new SQLite database connection `sqlite3.connect()` on every `get_cursor` context block incurs a massive performance penalty. In a test simulating 10,000 queries, dynamically connecting each time took ~0.75 seconds while re-using a connection took ~0.05 seconds (a 15x improvement). However, due to SQLite concurrency behavior, directly sharing a single connection across all threads can lead to thread-safety issues.
**Action:** Use `threading.local()` to cache a persistent connection per thread. Modify `DatabaseManager._get_conn()` to store and return `self._local.conn`. Then modify `get_cursor()` to reuse `self._get_conn()` instead of calling `sqlite3.connect()` repeatedly. This ensures each thread has its own connection object, preventing cursor leaks while avoiding connection-recreation overhead.
