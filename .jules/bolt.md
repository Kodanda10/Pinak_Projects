## 2024-05-10 - Thread-safe batch context managers
**Learning:** In highly concurrent applications like this FastAPI service, implementing a context manager like `VectorStore.batch_add` using a naive class lock for the duration of the yield can cause deadlocks if inner methods also try to acquire the lock. However, tracking batch state in standard instance variables (e.g. `self.in_batch`) creates race conditions across concurrent requests. Furthermore, repeated `np.vstack` calls during a loop cause an $O(N^2)$ slowdown.
**Action:** When implementing batching context managers in multi-threaded environments, use `threading.local()` to isolate the batch state buffer (like accumulating vectors in a list) to the specific thread. At the end of the batch, perform a single $O(N)$ `np.concatenate` operation while briefly acquiring the lock.
