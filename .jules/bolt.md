## 2025-08-29 - O(N^2) Performance Bug in NumPy Vector Store Batch Additions
**Learning:** When adding vectors incrementally in a loop (which is simulated by the existing `batch_add` context manager which only delays saving, not allocations), calling `np.vstack` repeatedly on large arrays results in O(N^2) performance because a new memory allocation and copy happens for every single vector added. This causes adding 20,000 vectors to take ~26 seconds.
**Action:** Use `threading.local()` to buffer vector additions into thread-safe Python lists during the context block. When the `batch_add` context manager exits, flush them to the main NumPy arrays with a single `np.vstack` and `np.concatenate`, changing the behavior to O(N) and bringing insertion time down to ~0.3 seconds.
