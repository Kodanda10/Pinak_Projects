## 2025-03-04 - VectorStore O(1) Amortized Inserts\n**Learning:** The initial implementation of the `VectorStore` used `np.vstack` and `np.concatenate` for every single vector insertion. This resulted in memory reallocation on every call, leading to O(N²) time complexity for sequential inserts. Replacing this with a dynamically resizing pre-allocated array (tracking `capacity` and `size`) achieved amortized O(1) time complexity.\n**Action:** Always avoid repeated memory allocations inside loops or frequent insert functions. Pre-allocate arrays with extra capacity and manually track the 'active' slice size, doubling capacity when the limit is reached.
