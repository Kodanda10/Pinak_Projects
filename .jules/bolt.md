## 2024-05-24 - [Thread Exhaustion in Debounced VectorStore Saves]
**Learning:** `threading.Timer.cancel()` does not prevent the OS thread from being created and starting execution. It only prevents the payload function from running. High-frequency debouncing using `threading.Timer` (where a new Timer is created and the old one cancelled repeatedly) leads to severe OS thread exhaustion, degraded system performance, and potential application crashes due to reaching thread limits.
**Action:** Instead of debouncing saves by constantly recreating and cancelling timers, use throttling: check `if self._save_timer is None:` before creating a new timer. Ensure the timer variable is securely reset in a `finally` block within the protected execution context to avoid missed executions or race conditions.
