## 2024-04-03 - [Optimize Numpy L2 Distance Search in Vector Store]
**Learning:** In `VectorStore.search()`, calculating L2 distances in NumPy using `.flatten()` on intermediate matrix operations (`query_vector.T`) causes overhead through allocations and transpositions. Since query vectors act essentially as 1D vectors, a fully vectorized approach using `.ravel()` for a 1D view, `np.dot(1D, 1D)` for norms, and in-place array modifications (`-=` and `out=`) drastically improves performance.
**Action:** When implementing or optimizing custom vector stores in pure NumPy without FAISS, prefer 1D vector operations (`.ravel()`) and strictly use in-place mathematical functions (`np.add(out=)`, `np.maximum(out=)`, `-=`) to prevent large intermediate array allocations during distance calculations across entire vector matrices.
