## 2024-06-25 - [Optimize NumPy L2 distance calculation in VectorStore.search]
**Learning:** Calculating L2 distances in NumPy using `.flatten()` and intermediate array allocations causes significant overhead. A fully vectorized approach using `.ravel()` for a 1D view, 1D `np.dot` for norms, and in-place array modifications (`-=` and `out=`) drastically improves performance. To prevent `UFuncTypeError` during in-place operations, ensure the 1D query array is explicitly cast to the target array's dtype (e.g., `.astype(self.vectors.dtype, copy=False)`).
**Action:** When working with large NumPy arrays, avoid intermediate memory allocations during basic operations. Use `.ravel()` to get a flat 1D view of queries, and apply in-place math operations like `-=` or `.out=` parameters wherever possible to eliminate temporary array overhead. Be explicit about dtype casting before performing in-place updates.
