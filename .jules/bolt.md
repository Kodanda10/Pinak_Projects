## 2024-05-09 - VectorStore.batch_add is O(N^2)
**Learning:** `VectorStore.batch_add` context manager yields without buffering, meaning `add_vectors` repeatedly calls `np.vstack` for every single addition during a batch operation. Since `np.vstack` allocates a new array each time, adding N vectors incrementally is O(N^2) complexity and extremely slow (taking ~3 seconds for just 5000 vectors).
**Action:** Use `threading.local()` to buffer vector additions in thread-safe Python lists during the context block. Then, when the context manager exits, flush them with a single `np.vstack` and `np.concatenate`. This brings the performance down to O(N) (from 3.1 seconds to ~0.09 seconds for 5000 additions). Ensure the commit logic is in the `try` block, not `finally`, so partial data isn't committed on exceptions, and properly handle nested contexts.
