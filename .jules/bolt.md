## 2026-04-24 - Missing Composite Indexes on Memory Tables cause O(N) Table Scans
**Learning:** The SQLite database `DatabaseManager` isolates queries heavily using `tenant`, `project_id`, and `client_id`, but the core tables like `memories_semantic` and `memories_episodic` lacked composite indexes. Benchmarks show this caused simple isolation queries and `get_client_layer_stats` to run O(N) full table scans, resulting in ~1s execution times for 100 queries on 50k rows. Adding composite indexes `(tenant, project_id, client_id)` dropped the latency to ~0.005s.
**Action:** When working on DB isolation (tenant/project scoping) in SQLite schemas, always verify if appropriate composite indexes are in place. When adding indexes, always remember to update `test_doctor_backfill.py` mock schemas as they tightly couple to the structure in `database.py`.
