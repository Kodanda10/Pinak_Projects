## 2025-03-05 - Mitigated SQL Injection & Mass Assignment in memory updates
**Vulnerability:** The `update_memory` method in `DatabaseManager` was iterating through dictionary keys and building a SQL set string without validating the column names (`set_clause = ", ".join([f"{k} = ?" for k in serialized.keys()])`). This permitted attackers to potentially inject logic via key string. Furthermore, `MemoryService.update_memory` allowed users to mass assign columns on valid tables without explicit allow-listing per memory layer (except explicitly forbidden ones like `id`, `tenant`).
**Learning:** Raw dictionaries obtained from user payloads MUST be validated for valid identifiers if their keys are to be used for SQL queries, even if values are parameterized. Mass assignment risks occur when API layers accept any schema keys instead of an explicit allow-list.
**Prevention:** Use an explicit allow-list per semantic layer (`ALLOWED_UPDATES`) to filter payload dicts early. In DB layer code, enforce column names with `isinstance(key, str) and key.isidentifier()`.
