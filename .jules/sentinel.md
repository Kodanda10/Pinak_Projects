## 2024-03-03 - [CRITICAL] Prevent SQL Injection and Mass Assignment via unvalidated keys in DatabaseManager.update_memory
**Vulnerability:** The `DatabaseManager.update_memory` method dynamically constructed SQL queries using dictionary keys (`set_clause = ", ".join([f"{k} = ?" for k in serialized.keys()])`). This permitted attackers to use malicious dictionary keys to inject arbitrary SQL statements or update unintended columns (Mass Assignment).
**Learning:** Even if values are properly parameterized (using `?`), the structure of the SQL query itself (like table or column names) must never be derived from untrusted input without strict validation. Python's `isidentifier()` is a highly effective way to validate column names.
**Prevention:** Enforce strict type checking and formatting on all user-controlled data that forms the structure of a SQL query. In Python, ensure keys are strings and use `.isidentifier()` to verify they are valid alphanumeric identifiers before interpolating them into SQL.
