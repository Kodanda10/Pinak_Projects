## 2024-05-17 - Fix Hardcoded Fallback JWT Secrets
**Vulnerability:** A pervasive vulnerability was discovered where core components of the memory service (CLI, ingest scripts, MCP client, and startup scripts) would fallback to default hardcoded JWT secrets like `"dev-secret-change-me"` or `"secret"` if `PINAK_JWT_SECRET` was omitted from the environment.
**Learning:** This fallback was meant for "developer convenience", but allowed systems to run insecurely in a deployed environment without throwing errors. Consequently, it created instances of weak authentication endpoints that could have been compromised because anyone could guess the `"secret"` fallback and forge JWTs.
**Prevention:** In order to prevent this, the code must adhere to the secure-by-default principle: no default secrets should be defined in source. If a required secret or configuration variable is not provided to the environment, the application or script must fail early (e.g. `sys.exit(1)`, raise `RuntimeError`), rather than substituting a default value.
