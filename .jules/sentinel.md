## 2024-05-24 - [SQL Injection via Dictionary Keys in Dynamic SQL UPDATE]
**Vulnerability:** In `app/core/database.py`'s `update_memory` function, the `updates` dictionary keys were being directly injected into the SQL `UPDATE` statement string without validation. Because the SQLite engine parameterized only the *values* and not the *keys*, an attacker could inject arbitrary SQL into the `UPDATE` query by crafting malicious keys (e.g., `"content = 'hacked' --"`).
**Learning:** Python DB API parameterization (like `execute("...", params)`) only protects the values passed to placeholders (`?`), not identifiers like column names. Dynamically constructing SQL using user-provided dictionary keys is an overlooked injection vector that bypasses traditional parameterization.
**Prevention:** Never trust user input for column names or identifiers in dynamic SQL construction. Always iterate through a dictionary and validate the key structure (`isinstance(key, str) and key.isidentifier()`) before using it in the query string.
