## 2024-05-24 - Fix SQL Injection via Dynamic Dictionary Keys in DatabaseManager.update_memory
**Vulnerability:** A potential SQL injection vulnerability existed in `DatabaseManager.update_memory` where arbitrary user-controlled dictionary keys from the `updates` payload were dynamically interpolated directly into an `UPDATE` query's `SET` clause using f-strings (`f"{k} = ?"`).
**Learning:** Python dictionary keys are inherently strings, but when constructing dynamic SQL columns, parameterization (e.g., `?`) only applies to values, not column names. Thus, formatting unchecked dictionary keys directly into SQL syntax creates a severe injection risk, even if the payload's source seems trusted, as mass assignment bugs or upstream validation bypasses could allow arbitrary SQL commands to execute.
**Prevention:** Always strictly validate dynamic column names against an allowed-list or use `key.isidentifier()` to guarantee they are valid, safe alphanumeric string literals before formatting them into SQL structures. Apply `# nosec` exclusions only on the specific lines where safe dynamic SQL is constructed, provided the upstream sanitization is visibly documented.
