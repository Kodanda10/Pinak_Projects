## 2025-03-25 - SQL Injection via Dictionary Keys in Dynamic SQL
**Vulnerability:** The `DatabaseManager.update_memory` method constructed the `SET` clause of a SQL `UPDATE` statement dynamically using dictionary keys from user-provided input. While the values were properly parameterized (`?`), the keys themselves were directly interpolated into the SQL string without validation. This allowed an attacker to inject arbitrary SQL commands by passing malicious string keys (e.g., `{"content = ?; DROP TABLE memories_semantic; --": "hacked"}`).
**Learning:** Parameterization only protects SQL values, not identifiers like column names or table names. Any user-controlled input used to construct SQL identifiers is an injection vector, even if the primary data values are parameterized.
**Prevention:** When dynamically constructing SQL identifiers from user input, such as dictionary keys, always validate that the input conforms strictly to the expected format. For column names, asserting that the string is a valid Python identifier (`isinstance(key, str) and key.isidentifier()`) provides a robust defense against injection without requiring expensive database schema lookups or hardcoding column lists that can cause tests to fail.
