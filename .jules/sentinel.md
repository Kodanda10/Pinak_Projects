## 2025-04-07 - Dynamic SQL Identifiers in DatabaseManager.update_memory
**Vulnerability:** SQL Injection (CWE-89) and mass assignment through dynamically constructing SQL using user-provided dictionary keys in `DatabaseManager.update_memory`. Python DB API parameterization protects values, but not identifiers like column names.
**Learning:** Even though we don't expose this API directly to users and it has denylists, allowing dictionary keys directly into f-strings for column names is a critical injection vector. When building dynamic `SET` clauses or `WHERE` clauses, the keys themselves must be validated. Using `isinstance(key, str) and key.isidentifier()` is a robust, schema-agnostic way to prevent SQL injection in identifier context without side effects or test failures caused by strict schema checks.
**Prevention:** Always sanitize dynamically constructed SQL identifiers (e.g., column names). Relying only on parameterization (`?`) for values is insufficient if the keys themselves are user-controlled. Enforce strict `isidentifier()` checks on dictionary keys before using them in SQL construction.
