## 2025-02-14 - Fix SQL injection via dynamic dictionary keys
**Vulnerability:** A SQL injection vector was present in `app/core/database.py` because the `DatabaseManager.update_memory` dynamically constructed `UPDATE` query statements by directly appending keys from an updates dictionary using an f-string: `set_clause = ", ".join([f"{k} = ?" for k in serialized.keys()])`. Since Python DB API parameterization only sanitizes values (not column names), this left the application vulnerable if an attacker could inject arbitrary keys into the update dictionary.
**Learning:** Python DB API (`sqlite3.execute(query, params)`) parameterization `?` only protects values, not identifiers like column or table names. Using user-provided dictionary keys directly to construct SQL statements can introduce critical SQL injection vulnerabilities, even if the values correspond to safe parameterization.
**Prevention:** Always validate and sanitize user-provided keys when dynamically building SQL column identifiers. Use `isinstance(key, str) and key.isidentifier()` to ensure the input keys are safe Python variables/column names, or implement a strict explicit allowlist.
