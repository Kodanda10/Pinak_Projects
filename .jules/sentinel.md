## 2025-02-14 - Fix SQL Injection in DatabaseManager.update_memory
**Vulnerability:** The `DatabaseManager.update_memory` method dynamically constructed a SQL `UPDATE` statement using dictionary keys from the `updates` payload without validation. This allowed potential SQL injection through column names, as Python's DB API parameterization only protects values passed to placeholders (`?`), not identifiers.
**Learning:** Dynamically constructing SQL queries using user-provided dictionary keys as identifiers is an injection vector, even if the values are parameterized. While there's a denylist check in the caller (`MemoryService.update_memory`), passing unchecked keys down to the query execution layer breaks defense-in-depth and is unsafe.
**Prevention:** Always validate user-provided keys when they are used dynamically as SQL identifiers. Enforcing `isinstance(key, str) and key.isidentifier()` provides a strict, built-in validation mechanism to ensure the keys are safe to be injected as column names, mitigating SQL injection without adding active DB schema checks or side effects.
